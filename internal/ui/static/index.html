How it works

Three steps. Then it stays out of your way.

No new chat interface to learn. NeuroFS sits between your editor and your model, building the right context behind the scenes.

01 — Describe

Tell it what you're doing

A sentence is enough. NeuroFS maps your intent to the right files using your repo's structure and your past sessions.

02 — Pack

A focused context is built

Irrelevant files are dropped. What remains is a compact, model-ready pack that fits comfortably in any context window.

03 — Resume

Pick up where you left off

A persistent journal preserves what changed and why. Tomorrow's session starts with yesterday's context — already loaded.

Guide

      

        
Pipeline

        
NeuroFS is a pipeline: scan → pack → (Claude) → replay. Scan builds the SQLite index at .neurofs/index.db. Pack ranks and compresses. Replay is the offline governance step.


        
Ranker signals

        
    
          
  • filename_match — token is in the path.
    • 
          
  • path_match — focus prefix hit.
    • 
          
  • symbol_match — token is in the symbols table (with shallow stemming).
    • 
          
  • import_match — token appears in imports.
    • 
          
  • focus / changed — boosts from --focus / --changed.
    • 
        


        
Audit metrics

        
    
          
  • grounded_ratio — fraction of citations that resolve to a file inside the bundle.
    • 
          
  • drift_rate — unknown / (known + unknown) symbols, split across three buckets:
            
      
              
    • paths — file-like references not in the bundle.
      • 
              
    • apis — dotted names (jwt.sign) not in the bundle.
      • 
              
    • symbols — plain code-shaped identifiers not in the bundle.
      • 
            
    
          
    • 
          
  • answer_recall — fraction of --facts strings present in the response (case-insensitive substring).
    • 
        


        
What is implemented

        
    
          
  • Local HTTP server + embedded UI (internal/ui).
    • 
          
  • Endpoints wrap scan, pack, replay, records list, diff, stats.
    • 
          
  • Bundle is held in memory between pack and replay. Use snapshot path for cross-session replay.
    •
